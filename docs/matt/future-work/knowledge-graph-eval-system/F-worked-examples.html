U13. Worked example cases (one per target type)

Goal: Ship one end-to-end example case per target kind as fixtures, including a poison case, proving the harness runs end-to-end.
Requirements: R2, R6 (entry points), exercises U1–U10  ·  Dependencies: U10
Files: evals/cases/<code-case>/…, evals/cases/<doc-case>/…, evals/cases/<tickets-case>/…, evals/cases/<poison-case>/…, evals/tests/test_examples_e2e.py

Approach: Author a small code-target case (against a real pinned GitHub or GitLab repo+commit), a requirements-doc case, a ticket-list case, and a poison/negative-control case (an ignored conflicting ingestion). Each includes ingestions, seed prompt, target, frozen rubric where applicable, and an assertion. An e2e smoke test runs them under FakeRunner so CI verifies the whole pipeline deterministically; the real-runner path is exercised manually/integration.

Patterns to follow: these fixtures are the template for all future cases and for later bootstrapped cases (R10).

Test scenarios:

  • E2e (fake): each example case runs cold+injected → grade → attribute → assert → scoreboard without error.
  • Poison case: the negative-control case passes only when the poison is absent from the generated CLAUDE.md.
  • Schema: every example case validates against U1.
  • Sufficiency sanity: each helpfulness example is constructed so cold under-performs and injected improves under FakeRunner scripting (encodes the origin sufficiency intent).
