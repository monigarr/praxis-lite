Design decisions

Phase-1 substrate is a CLAUDE.md file; reader is a whole-file tool (KTD-1). The "knowledge graph" is a generated CLAUDE.md; the "reader" is a registered Claude tool whose function returns the entire file. No ranking, no selective retrieval yet. The harness talks to a Reader/KnowledgeSubstrate protocol so Phase-2 retrieval swaps in behind the same seam (see origin: Deferred — substrate bake-off).
Write-time poison resistance (KTD-3, write-time half). Phase-1 poison-resistance asserts the poison insight is absent from the generated CLAUDE.md (write-time credibility/conflict-resolution), since the whole-file reader cannot exclude at read time. A code comment + schema note records that this assertion returns to retrieval-level in Phase 2. (The assertion-evaluation half of KTD-3 lives in Component E / U9.)

U3. Phase-1 knowledge substrate and reader tool

Goal: Generate a CLAUDE.md from a case's used ingestions and expose a whole-file reader tool, behind a swappable substrate/reader seam.
Requirements: R3 (seam for later retrieval), KTD-1, KTD-3  ·  Dependencies: U1
Files: evals/knowledge/base.py, evals/knowledge/claude_md.py, evals/knowledge/read_tool.py, evals/tests/test_claude_md.py

Approach: Define KnowledgeSubstrate (build from ingestions → produce injected artifact) and Reader (the tool surface the agent calls) protocols. Phase-1 ClaudeMdSubstrate distills used ingestions into a CLAUDE.md (distillation itself may be a thin pass for Phase 1 — generation is the seam, not the research problem). read_tool registers a Claude tool whose function returns the whole CLAUDE.md verbatim and records that it was called (for attribution, U7). ignored-role ingestions are routed through the same credibility/conflict-resolution path so poison cases can assert absence from the output CLAUDE.md (KTD-3). Keep the protocol minimal so a Phase-2 retrieving reader implements the same interface.

Patterns to follow: protocol-based seams; the read tool mirrors the production injection mechanism named in docs/proposal-praxis.md §Inject & retrieve.

Test scenarios:

  • Happy path: used ingestions produce a non-empty CLAUDE.md containing their lessons.
  • Whole-file read: the reader tool returns the exact file contents and flags itself called.
  • Poison exclusion: an ignored/conflicting ingestion is absent from the generated CLAUDE.md (the Phase-1 resistance signal).
  • Seam: a stub alternate Reader satisfies the protocol and is accepted by the orchestrator (compile/contract test).
  • Empty: a case with no used ingestions yields an empty/absent artifact and the injected arm degenerates to cold (asserted in U8).
